# Git Commit with AIGC Tag

## 公司规范说明

根据质量管理部《关于统一AI代码占比统计规范的通知》（2026年4月10日），AI代码占比统计统一采用提交日志标注方式执行。

### AIGC标记要求（必填）

**强制要求**：AI生成代码提交配置库时，须在提交日志（Commit Message）中添加大写标记 `AIGC`，**位置不限**。

### Commit Message 格式

**基本格式：**

```
<需求/Bug编号>：<功能说明>。AIGC
```

**带采纳率格式：**

```
<需求/Bug编号>：<功能说明>。AIGC，采纳率XX%
```

**格式要求：**

- `AIGC` 必须大写
- 需求/Bug编号后使用中文冒号 `：`
- 功能说明后使用中文句号 `。`
- AIGC标记位置在句号之后
- 采纳率为可选项，格式为 `采纳率XX%`（采纳率和XX%间无空格）
- **每次提交都必须询问是否需要标注采纳率**

### 官方示例

**示例1：基本格式**

```
ZQSPB-8954：补充网络投票上场文件。AIGC
```

**示例2：带采纳率**

```
ZQSPB-8954：补充网络投票上场文件。AIGC，采纳率80%
```

## 使用流程

### 步骤1：确认编号和描述

在生成commit message前，主动询问用户：

- 需求/Bug编号是什么？（如 ZQSPB-8954）
- 功能说明是什么？（如 补充网络投票上场文件）
- **必须询问**：是否需要标注采纳率？

### 步骤2：分析变更内容

检查 `git diff --staged` 或 `git status` 了解本次提交的内容，确保功能说明准确。

### 步骤3：自动计算采纳率（如果需要）

如果用户选择标注采纳率，自动计算本次AI生成代码的占比：

- 分析变更的代码行数
- 计算AI生成代码占比
- 生成交付率百分比

### 步骤4：生成Commit Message

根据以下规则生成：

1. **如果代码由AI生成** → 使用格式：`<编号>：<功能说明>。AIGC`
2. **如果需要标注采纳率** → 使用格式：`<编号>：<功能说明>。AIGC，采纳率XX%`
3. **如果代码非AI生成** → 按原有规范，不添加AIGC标记

### 步骤5：确认并执行

向用户展示生成的commit message，确认后执行提交。

## 常见场景

### 场景1：AI生成代码（基本格式）

```
需求编号：ZQSPB-8954
功能说明：补充网络投票上场文件
采纳率：不标注
生成commit：ZQSPB-8954：补充网络投票上场文件。AIGC
```

### 场景2：AI生成代码（带采纳率）

```
需求编号：ZQSPB-8954
功能说明：补充网络投票上场文件
采纳率：自动计算为85%
生成commit：ZQSPB-8954：补充网络投票上场文件。AIGC，采纳率85%
```

### 场景3：非AI代码

```
需求编号：ZQSPB-8955
功能说明：优化页面加载性能
生成commit：ZQSPB-8955：优化页面加载性能
```

### 场景4：Bug修复（AI生成）

```
Bug编号：ZQSPB-8956
功能说明：修复表单验证逻辑错误
采纳率：不标注
生成commit：ZQSPB-8956：修复表单验证逻辑错误。AIGC
```

## 注意事项

1. **必须主动询问编号和功能说明**：不要猜测或编造需求/Bug编号
2. **每次必须询问是否标注采纳率**：提供选项让用户选择
3. **格式规范**：编号后用中文冒号 `：`，功能说明后用句号 `。`
4. **AIGC必须大写**：统一使用大写 `AIGC`
5. **自动计算采纳率**：如果用户选择标注，自动计算AI代码占比
6. **采纳率格式**：`采纳率XX%`（无空格）

## 统计说明

质量管理部将依据AIGC标识开展AI代码占比统计：

- **必填**：AIGC标记（用于统计AI代码占比）
- **可选**：采纳率标注（用于统计AI代码采纳率）

如有其他统计需求可联系质量管理部协调支持。

## 判断是否为AI生成代码

以下情况视为AI生成代码，需添加AIGC标记：

- Agent工具直接生成或修改的代码
- AI辅助编写的代码片段
- AI建议并实施的代码重构
- AI生成的配置文件

以下情况无需AIGC标记：

- 用户自行编写的代码
- 仅由AI提供咨询建议，人工编写代码
