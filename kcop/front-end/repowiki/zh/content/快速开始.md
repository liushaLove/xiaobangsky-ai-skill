# 快速开始

## 1. 确认项目类型

当前项目根目录应包含 `loader.js`、`loader-config.js`、`loader-frame-config.js`、`Gruntfile.js`、`apps/`、`basic/`、`frame/`、`lib/`。

如果你看到 `src/main.js`、`vite.config.js`、`package.json` 作为主入口，那是另一个项目的知识库内容，不适用于这里。

## 2. 定位业务页面

```powershell
rg -n "页面标题|字段名|服务号|data-main" apps basic frame
```

找到 HTML 后看 `data-main`，例如 `data-main="./src/reset-cust-org"` 对应 `src/reset-cust-org.js`。

## 3. 修改页面逻辑

优先查看 `exports.$ready`、`flow.init`、`Backbone.View.initialize`、`validate`、`getData/save`。

典型请求：

```javascript
ajax.post({
    req: {
        service: "P9999999",
        bex_codes: "YGT_A1100807",
        CUST_CODE: custCode
    }
});
```

典型组件：

```javascript
this.$org.combotree("getValue");
this.$form.form("load", data);
this.$form.form("disable", ["CUST_CODE"]);
```

## 4. 修改模板

如果改的是 `apps/tpls/**/*.html`，执行 `build-tpls.bat` 或 `grunt clean:appsTpls`、`grunt tmod:apps`。

## 5. 构建

默认构建由 `Gruntfile.js` 定义，可通过 `build.bat` 或 `grunt` 触发。构建会生成 `dist/`，并按任务复制到 `../../kcop-dist/`。
