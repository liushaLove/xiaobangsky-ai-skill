# 业务模块

## 总体结构

当前 copweb 的业务模块按目录和子系统拆分，不是 `src/pages/aoi` 这种 Vue 模块树。主要入口在 `apps/opp`，并保留多个 `k*` 子系统目录。

## apps/opp 核心模块

| 模块 | 目录 | 说明 |
| --- | --- | --- |
| 综合入口 | `apps/opp/pages` | 首页、菜单、门户入口 |
| 柜员办理 | `apps/opp/operator` | 客户资料、业务受理、流程节点 |
| 管理配置 | `apps/opp/manager` | 参数、权限、机构、监控、业务配置 |
| 复核审核 | `apps/opp/review` | 审核、复核、影像、结果处理 |
| 清算结算 | `apps/opp/settlement` | 清算/结算类业务 |
| 公共能力 | `apps/opp/common` | service、模块、native、打印、公共页面 |

## 子系统目录

根目录下的 `kcop`、`kuas`、`kidm`、`kisc`、`kgis`、`kgob`、`kmos`、`kfms`、`kfis`、`kdrf`、`kscs`、`kspb`、`kone`、`kidh`、`cicc` 等目录存放子系统兼容页面、静态资源或业务模块。
