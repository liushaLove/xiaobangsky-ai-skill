# 应用栏组件

> 本页已从原 fs-aoi-web(Vue/Vite) 知识库改写为当前 copweb 项目口径。当前项目是 KJDP4/KUI legacy 前端，不使用 Vue、Vite、Pinia、Vue Router 或 `@szkingdom.kjdp/core` npm 包。

## 当前项目定位

copweb 是以 `loader.js` 为入口、以 AMD/CMD 风格 `define(function(require, exports, module){ ... })` 为模块组织方式的多子系统静态前端工程。核心运行时由 `loader-config.js`、`loader-frame-config.js`、`frame/core.js`、`frame/kui.js`、`basic/ready.js` 串联，业务页面主要分布在 `apps/opp/**`、`basic/**` 和各 `k*` 子系统目录。

## 维护准则

- 页面入口看 HTML 的 `data-main`，再进入对应 `src/*.js`。
- JS 模块沿用 `define(function(require, exports, module){ ... })`。
- UI 沿用 KUI jQuery 插件和 `kui-options`。
- 请求沿用全局 `ajax.post({ req: { service: ... } })` 或 service 封装。
- 构建沿用 Grunt 与 `.bat` 脚本。

## 本页在 copweb 中的对应关系

当前 copweb 的门户不是 Vue Portal，而是 apps/opp 入口页、frame/views/content-view.js、frame/views/tabs-view.js、loader 和 KUI 共同组成的页面框架。
