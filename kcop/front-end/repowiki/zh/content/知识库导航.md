# copweb 知识库导航

本目录已按 `F:\git\kcop-zh-project\aoi\20250721\copweb` 重新梳理。原文档多数来自 `fs-aoi-web` 的 Vue/Vite 工程，已统一迁移为当前 copweb 的 KJDP4/KUI legacy 前端口径。

## 推荐阅读顺序

1. `项目概述.md`：先理解项目定位和运行链路。
2. `技术栈.md`：确认不要按 Vue/Vite 思路处理。
3. `项目结构详解.md`：定位目录职责。
4. `快速开始.md`：查找页面、修改页面、验证改动。
5. `kjdp-core-api.md`：当前项目真实 API，是全局 `ajax/kui/utils` 与 KUI 插件。
6. `API参考/组件API.md`、`API参考/服务接口.md`：改业务页最常用。
7. `构建与部署/构建配置.md`：Grunt、tmod、dist/kcop-dist。
8. `开发指南/代码规范.md`：兼容性和代码风格红线。

## 当前项目一句话模型

`index.html` 跳转到 `apps/opp/index.html`，页面 HTML 通过 `loader.js` 和 `data-main` 加载 AMD/CMD 模块；`loader-config.js` 和 `loader-frame-config.js` 装配 KJDP4/KUI 框架；`basic/ready.js` 挂载业务公共能力；页面 JS 使用 jQuery、Backbone、KUI 插件和 `ajax.post` 调后端服务；构建由 Grunt 复制、transport、concat、uglify、cssmin、tmod 完成。
