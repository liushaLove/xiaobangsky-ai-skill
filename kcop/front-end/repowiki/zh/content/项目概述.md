# 项目概述

## 项目定位

copweb / kcop 是一套 KJDP4/KUI 体系下的证券业务综合运营前端静态资源工程。它不是现代 Vue 单页应用，而是通过 `loader.js` 动态加载 HTML 页面和 AMD/CMD 风格 JS 模块的 legacy 多子系统前端。

核心入口和配置：

- `index.html`：根入口，跳转到 `apps/opp/index.html`。
- `loader.js`：页面模块加载器。
- `loader-config.js`：项目级配置，`proName: "FSAOI"`，`frameReady: "basic/ready-kdop"`。
- `loader-frame-config.js`：框架级配置，KUI 版本 `4.0.3`，加载 `frame/lib`、`frame/core`、`frame/kui`。
- `basic/ready.js`：业务基础初始化，加载公共工具、服务、缓存和 KUI 默认行为覆盖。
- `Gruntfile.js`：构建、压缩、CMD 依赖处理、模板编译与发布拷贝。

## 运行链路

1. 用户访问根页面或子系统入口 HTML。
2. HTML 引入 `loader.js`，通过 `data-main` 指向页面主模块。
3. loader 读取项目配置和框架配置，加载 KUI 框架层。
4. `basic/ready.js` 初始化 `kcopTop.kui`、缓存、字典、机构、公共 service。
5. 页面主模块的 `exports.$ready` 执行业务初始化。
6. 页面通过 KUI jQuery 插件渲染表单、表格、弹窗等组件，通过 `ajax.post` 或服务封装调用后端。

## 与 fs-aoi-web 的关键差异

| 维度 | fs-aoi-web | 当前 copweb |
| --- | --- | --- |
| 应用形态 | Vue 3 SPA | 多 HTML 页面 + loader 动态模块 |
| 构建工具 | Vite | Grunt + bat 脚本 |
| 模块语法 | ESM / Vue SFC | AMD/CMD `define(require...)` |
| 路由 | Vue Router | 菜单/页面 URL/iframe/标签页体系 |
| 状态 | Pinia | 顶层 `kcopTop.kui`、缓存、Backbone Model、本地变量 |
| UI | KJDP Vue UI | KUI jQuery 插件 |
| 请求 | `@szkingdom.kjdp/core` request | 全局 `ajax` / `ajax.post` / service |

## 维护原则

- 从 HTML 的 `data-main` 定位页面 JS。
- 先复用当前页面附近已有模块和公共 service。
- 不引入 Vue/Vite/ESM/TypeScript。
- 不批量格式化 legacy 文件。
- 改模板后运行 `build-tpls.bat` 或 `grunt tmod:apps`。
- 公共框架层改动需谨慎扩大回归范围。
