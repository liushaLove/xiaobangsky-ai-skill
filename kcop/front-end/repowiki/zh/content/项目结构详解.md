# 项目结构详解

## 根目录

```text
copweb/
├─ apps/                  # 主要业务页面与子应用
├─ basic/                 # 基础业务公共层
├─ frame/                 # KJDP/KUI 框架层
├─ lib/                   # 第三方库与 KUI/web2 组件库
├─ upload/                # Excel/DBF 模板、业务视频图片等静态资源
├─ kcop/ kuas/ kidm/ ...  # 多子系统静态资源或兼容页面
├─ loader.js              # 加载器
├─ loader-config.js       # 项目配置
├─ loader-frame-config.js # 框架配置
├─ Gruntfile.js           # 构建配置
└─ build*.bat             # 构建/模板/发布脚本
```

## apps/opp

| 目录 | 职责 |
| --- | --- |
| `pages/` | 综合运营入口页、菜单、首页等 |
| `operator/` | 柜员业务办理、资料录入、客户相关节点 |
| `manager/` | 参数、权限、机构、业务配置等管理页面 |
| `review/` | 复核、审核、影像核查等页面 |
| `settlement/` | 清算/结算相关页面 |
| `common/` | opp 公共模块、service、native、打印、影像等 |
| `frame/` | opp 门户框架资源、登录页、logo、公共视图 |

页面通常成组出现：

```text
some-dir/
├─ some-page.html
├─ src/some-page.js
└─ css/some-page.css
```

## basic

`basic/ready.js` 是入口初始化核心。`basic/modules` 提供业务工具、字典、机构、菜单、参数模型；`basic/service` 提供客户、员工、流程、影像、短信、系统等公共 service；`basic/views` 提供复核、影像、打印、导入等公共视图。

## frame

`frame/core.js` 挂载核心对象，`frame/core/ajax.js` 封装请求协议，`frame/kui.js` 引入 KUI 组件，`frame/views` 实现标签页和内容区，`frame/themes` 提供主题样式。

## lib 与 upload

`lib/kui`、`lib/kui/web2` 是 KUI 组件实现和兼容层。`upload/excel-tpl`、`upload/dbf-tpl` 存放导入导出模板，文件名往往与业务代码和后端配置绑定。
