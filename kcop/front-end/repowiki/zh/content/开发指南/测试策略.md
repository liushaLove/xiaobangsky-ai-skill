# 测试策略

仓库没有现代单元测试体系，`Gruntfile.js` 中保留 `qunit` 任务但业务开发主要依赖静态检查、构建和页面手工验证。

| 改动 | 验证 |
| --- | --- |
| JS 页面逻辑 | 检查 `define`、依赖路径、括号、页面手工打开 |
| HTML 字段 | 检查 JS 选择器、form load/save、组件校验 |
| CSS | 检查页面布局和 KUI 弹层/表单宽度 |
| tmod 模板 | 运行 `build-tpls.bat` |
| ajax/service | 验证成功、失败、空数据、会话超时 |
| 公共层 | 至少回归多个入口页和典型业务页 |
