# 代码规范

## 语法兼容

- 沿用 ES5 风格：`var`、普通函数、字符串拼接。
- 不要引入 TypeScript、ESM、Vue SFC、`async/await` 等需要转译的写法。
- 避免可选链、空值合并、class fields 等新语法。

## 模块规范

```javascript
define(function (require, exports, module) {
    var mod = require("./mod");
    exports.$ready = function () {};
});
```

## 风格

- 缩进跟随周边文件，多数为 4 空格。
- 引号风格跟随周边，不做机械统一。
- 不批量格式化 legacy 文件。
- 不随意改全局变量名和组件 id/name。
