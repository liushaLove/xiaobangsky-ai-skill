# Git 提交 AIGC 标记

## 规范

AI 生成或 AI 辅助修改的代码提交，commit message 必须包含大写标记 `AIGC`。如果用户要求标注采纳率，在 `AIGC` 后追加 `，采纳率XX%`。

## 格式

```text
<需求或 Bug 编号>：<功能说明>。AIGC
<需求或 Bug 编号>：<功能说明>。AIGC，采纳率XX%
```

要求：

- `AIGC` 必须大写。
- 编号后使用中文冒号 `：`。
- 功能说明后使用中文句号 `。`。
- 不要编造需求或 Bug 编号。
- 每次提交前都询问用户是否需要标注采纳率。

## 工作流

1. 查看 `git status --short` 和待提交 diff，确认变更范围。
2. 向用户确认需求/Bug 编号、功能说明、是否标注采纳率。
3. 生成提交信息并展示给用户确认。
4. 用户确认后再执行 `git commit`。

## 示例

```text
ZQSPB-8954：补充网络投票上场文件。AIGC
ZQSPB-8954：补充网络投票上场文件。AIGC，采纳率80%
```

## 是否需要 AIGC

需要标注：

- Agent 工具直接生成或修改的代码、配置、文档。
- AI 建议并由 Agent 实施的重构或修复。

通常不需要标注：

- 用户自行编写的代码。
- AI 仅提供咨询建议，没有参与实际代码产出。
